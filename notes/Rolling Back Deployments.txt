Rolling Back Deployments


Rolling update revisions can be tracked

If a Deployment has issue, a new Deployment can be applied, or you can revert to a previous revision

Several kubectl commands can be used for rollbacks

- k rollout status -f file.deployment.yml
- k rollout history 
- k rollout undo -f file.deployment.yml <=== rollback to the previous deployment

# Get information about a Deployment
k rollout history deployment [deployment-name]

# Get information about a Deployment
k rollout history deployment [deployment-name] --revision=2 <=== jump back to a history item or back multiple versions


Some companies don't roll back and just fix the issue instead of rolling back


k rollout history deployment my-nginx

k rollout undo deployment my-nginx
