ConfigMaps

ConfigMaps provide a way to store configuration information and provide it to containers.


     -----------------       
    |                |       
    |                |       - Provides a way to inject configuration data into a container
    |     Pod        |       - Can store entire files or provide key/value pairs:  
    |   ----------   |         Store in a File. Key is the filename, value is the file contents 
    |  |          |  |         (can be JSON, XML, keys/values, etc.)
    |  |          |  |       - Provide on the command-line
    |  | Container|  |       - Use YAML, ConfigMap manifest
    |  -----------   |      
    ------------------         
            |                
         ConfigMap  


Accessing ConfigMAP Data in a Pod

ConfigMaps can be accessed from a Pod using

  - Environment variables(key/value)
  - ConfigMap Volume (access as files)




     ------ Pod ------         ---- ConfigMap ----
    |                |         |                  |
    |                |         |                  |
    |                |         |                  |
    |   Environment  |-------->|   Key1=value 1   |
    |    Variables   |         |                  |
    |                |         |                  | 
    |    ConfigMap   |-------->|   Key2=value 2   |  
    |     Volume     |         |                  |
    ------------------         -------------------- 

Key command-line switches include"
--from-file
--from-env-file
--from-literal

k create configmap [cm-name] --from-file=[path-to-file]

k create configmap [cm-name] --from-env-file=[path-to-file]

k create configmap [cm-name] --from-literal=apiUrl=https://my-api --from-literal=otherKey=otherValue

k create -f file.configmap.yml

k get cm [cm-name] -o yaml


1. Using a k8s ConfigMap manifest

k create -f settings.configmap.yml

2. build image 

docker build -t node-configmap .

3. Create ConfigMap

k create cm app-settings --from-env-file=settings.env

k get cm

k get cm app-settings -o yaml


4. Create deployment

k apply -f node.deployment.yml

5. Port forward the Pod

k port-forward node-configmap-59b556fd98-9h7px 9000

6. curl http://localhost:9000
